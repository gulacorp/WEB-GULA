04
Versionado

Commit a GitHub: cada cambio queda registrado y reversible.

05
Publicación

GitHub Pages despliega automáticamente al hacer push.

06
Operación

Servicios externos procesan datos reales (pedidos, leads, analítica).

¿Qué pasa cuando hago un cambio?

1. Le explico a Cascade qué quiero cambiar (ej: "añade botón de WhatsApp en el footer"). 2. Cascade lee la base de código, propone el cambio archivo por archivo. 3. Reviso y apruebo. 4. Cascade ejecuta y crea un commit descriptivo. 5. Hago push a GitHub. 6. GitHub Pages compila y publica en ~2 minutos. 7. Verifico el resultado en producción.

Ningún paso requiere intervención manual de servidores. Todo es declarativo y reversible.

Lectura ejecutiva del mapa3 conclusiones

1. La estructura está montada — Las 15 páginas existen, navegan entre sí, y respetan estándares de marca. La web "parece" completa al visitante casual.

2. Los huecos críticos son backend, no diseño — Faltan: checkout funcional, sistema de puntos del Club, confirmación de pedido, dashboard franquiciado. Sin esto, no podemos cobrar online.

3. Lo legal y SEO está casi vacío — Los textos legales son placeholders. Falta sitemap, robots.txt, meta OG, banner cookies. Esto es bloqueante antes de hacer marketing serio.

4 · Stack tecnológico

Las 8 piezas que componen la web

Click para desplegar el detalle de cada capa.

Frontend (lo que ve el cliente)HTML · CSS · JS

HTML + CSS + JavaScript puro, sin frameworks pesados. Carga rápida, fácil de mantener, sin dependencias raras. 15 páginas estáticas optimizadas.

Backend (servidor)Node.js · Express

Recibe formularios, valida pagos con Stripe, envía emails y conversa con Supabase. Hoy corre local; en producción irá en Cloudflare Workers o Render.

Base de datosSupabase · Postgres

Postgres gestionado. Guarda usuarios del Club, pedidos, leads de franquicias, registros de eventos. Replicado y con copias automáticas.

PagosStripe · 1,5% + €0,25

Stripe procesa tarjetas; pedidos por agregadores (Uber Eats, Glovo, Just Eat) se redirigen a sus apps. Stripe gestiona reembolsos, suscripciones del Club y facturación.

Email transaccionalResend

Bienvenida, tickets, confirmaciones de pedido, alertas internas. Free tier 3.000 emails/mes.

Analítica de productoPostHog

Rastrea cada clic, cada flujo, cada abandono. Ver qué hacen los usuarios y qué convierte mejor. Cumple RGPD. Free tier 1M eventos/mes.

Hosting y CDNGitHub Pages · Cloudflare (futuro)

GitHub Pages publica gratis. Migración prevista a Cloudflare Pages + Workers cuando lleguemos a 5K usuarios para mejor rendimiento global.

Mapas y ubicaciónGoogle Maps Platform

Las cuatro tiendas con direcciones precisas. A futuro: cálculo de zonas de delivery propio y geolocalización del cliente. $200 crédito gratis/mes.

Ejemplo: cómo se procesa un pedido en 9 pasos

1. Cliente añade burgers al carrito (frontend) → 2. Navegador comprueba si está logueado en Club (localStorage) → 3. Si no, redirige a CLUBGULA.html → 4. Una vez logueado, valida el pago con Stripe → 5. Stripe confirma → 6. Backend guarda pedido en Supabase → 7. Resend envía email de confirmación → 8. PostHog registra conversión → 9. Cliente ve "Pedido confirmado". Todo en menos de 3 segundos.

Stripe — el coste variable real

Las comisiones de Stripe se calculan sobre el volumen de ventas. Cada €1.000 facturados online ≈ €18-22 a Stripe. Con ticket medio €25 y 10K pedidos/mes (€250.000 facturación): ~€4.500/mes en comisiones. Coste integrado en el producto, no es operativo del SaaS.
