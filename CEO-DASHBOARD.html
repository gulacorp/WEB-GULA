Hoja 02 · Cómo se construye

De la idea al deploy en 6 pasos

Flujo completo de desarrollo. Cada cambio se registra, se versiona y se publica automáticamente sin intervención manual de servidores.

~2min
Tiempo deploy
01
Idea

Indicación interna o métrica detectada. Se traduce en una tarea concreta.

02
Diseño

Aplicar reglas de STANDARDS.md (tipografías, colores, copy).

03
Desarrollo

Edición en Windsurf con asistente IA. Revisión humana antes de aplicar.

04
Versionado

Commit a GitHub. Cada cambio queda registrado y reversible.

05
Publicación

GitHub Pages compila y despliega automáticamente al hacer push.

06
Operación

Servicios externos procesan datos reales: pedidos, leads, analítica.

¿Qué pasa cuando se hace un cambio?

1. Le explico a Cascade qué quiero. 2. Cascade lee el código y propone el cambio. 3. Reviso y apruebo. 4. Cascade ejecuta y crea commit. 5. Push a GitHub. 6. GitHub Pages publica en ~2 minutos. Todo declarativo y reversible.

Hoja 05 · Stack tecnológico

Las 8 piezas que componen la web

Capas de la plataforma desde el navegador hasta los servicios externos. Cada una con su rol específico.

8
capas técnicas

Frontend

HTML · CSS · JS

HTML + CSS + JavaScript puro, sin frameworks pesados. Carga rápida, fácil de mantener. 17 páginas estáticas optimizadas.

Backend

Node.js · Express

Recibe formularios, valida pagos con Stripe, envía emails y conversa con Supabase. Hoy local; en prod irá a Cloudflare Workers o Render.

Base de datos

Supabase · Postgres

Postgres gestionado. 10 tablas: crew_members, pedidos, events, referrals, nps_responses, reservations, submissions, contactos, puntos_historial, promociones. 3 vistas: v_journey_funnel, v_members_by_store, v_submissions_summary. RLS activo.

Pagos

Stripe · 1,5% + €0,25

Stripe procesa tarjetas; pedidos por agregadores (Uber, Glovo) se redirigen a sus apps. Stripe gestiona reembolsos, suscripciones y facturación.

Email transaccional

Resend

Bienvenida, tickets, confirmaciones de pedido, alertas internas. Free tier 3.000 emails/mes con DX moderno.

Analítica

PostHog

Rastrea cada clic, cada flujo, cada abandono. Cumple RGPD. Free tier 1M eventos/mes.

Hosting + CDN

GitHub Pages → Cloudflare

GitHub Pages publica gratis. Migración prevista a Cloudflare Pages + Workers cuando lleguemos a 5K usuarios.

Mapas

Google Maps Platform

Las cuatro tiendas con direcciones precisas. A futuro: zonas de delivery propio y geolocalización del cliente. €200/mes gratis.

Cómo se procesa un pedido en 9 pasos

1. Cliente añade burgers al carrito. 2. Navegador comprueba si está logueado en Club. 3. Si no, redirige a CLUBGULA. 4. Pago con Stripe. 5. Stripe confirma. 6. Backend guarda pedido en Supabase. 7. Resend envía email confirmación. 8. PostHog registra conversión. 9. Cliente ve "Pedido confirmado". Todo en menos de 3 segundos.

Fortalezas y debilidades

Análisis honesto de qué se gana y qué hay que asumir al construirlo en casa

SWOT

Fortalezas

  • Coste marginal cercano a cero. Las primeras 1.000 conversaciones de servicio cada mes son gratis para Meta. Solo pagas Cloudflare Worker (€5).
  • Conversión brutal. WhatsApp tiene tasa de apertura del 98% (vs 22% del email). Mensajes leídos en <3 minutos.
  • Datos 100% propios. Sesión, historial, preferencias del cliente en tu Supabase. Reutilizable para email marketing y CRM.
  • Integración nativa con stack. Mismo Stripe, mismo Postgres, mismo Cloudflare ya activos. No añade complejidad operativa.
  • Sin lock-in. Cambiar de proveedor (Twilio, 360dialog) o pasar a otra arquitectura es solo cambiar las credenciales del API.
  • Escalable globalmente. Cloudflare Worker corre en 300+ ciudades, latencia <50ms. Aguanta picos de Black Friday sin tocar infraestructura.

Debilidades

  • Verificación Meta Business. Hay que pasar verificación de empresa con Meta (CIF, web pública, dueño legal). Tarda 1-3 semanas.
  • Plantillas pre-aprobadas. Para iniciar conversación con cliente, las plantillas las aprueba Meta (24-48h). Imposible improvisar campañas.
  • Marketing es de pago. Las conversaciones de marketing en España cuestan €0,06-0,08 por conversación. Hay que medir ROI.
  • Mantenimiento técnico. Es código propio: bugs, monitorización, actualizaciones. Necesita ~4-8h/mes de mantenimiento.
  • Riesgo de baneo. Si los clientes marcan mensajes como spam, Meta degrada la calidad y puede bloquear el número. Hay que cuidar el opt-in.
  • Sin UI para staff. Las plataformas tipo Crisp dan inbox visual. Aquí, para soporte humano, hay que conectar con Crisp/Chatwoot vía API.

Cuándo arrancarlo

Recomendación: arrancar el piloto cuando el Club GULA tenga 200+ socios. Antes no compensa el setup. Punto de inflexión: a los 1.000 socios, el bot empieza a generar más pedidos que las apps de delivery (Uber/Glovo) por menor fricción y CAC. ROI: cada €1 invertido en el bot devuelve €8-12 en pedidos directos al evitar comisiones de agregadores (25-30%).
